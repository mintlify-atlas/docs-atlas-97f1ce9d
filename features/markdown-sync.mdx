---
title: "Sync Zotero notes with Markdown files using Better Notes"
description: "Keep Zotero notes in two-way sync with external Markdown files. Set up auto-sync, manage synced notes, and review changes with the built-in diff tool."
---

Better Notes can keep any Zotero note in continuous two-way sync with an external Markdown file on your filesystem. Changes you make in the Zotero editor flow out to the `.md` file, and edits you make in an external Markdown editor like Obsidian flow back into Zotero. No third-party sync tools or scripts are required.

## How sync works

When a note is synced, Better Notes tracks both the Zotero note content and the linked `.md` file. After each sync cycle, it records a checksum for both sides. On the next sync, it compares the current state against those checksums to detect which side changed:

- If only the **Zotero note** changed, Better Notes writes the updated content to the `.md` file.
- If only the **Markdown file** changed, Better Notes reads the file and updates the Zotero note.
- If **both sides** changed since the last sync, Better Notes opens the diff window so you can review and merge the changes manually.

<Note>Sync runs after you close the note editor and again after the auto-sync period you set in preferences. It does not run continuously in the background while the editor is open.</Note>

## Setting up auto-sync

<Steps>
  <Step title="Export a note for the first time">
    Open a note as a tab. In the editor toolbar, open the **settings menu** (the gear icon) and click **Export**.
  </Step>
  <Step title="Choose Markdown with auto-sync">
    In the export options window, select **Markdown** as the format and enable **Set Auto-Sync**.
  </Step>
  <Step title="Select a sync folder">
    A folder picker opens. Choose or create the directory where you want the `.md` file to live. This is typically your Obsidian vault or another Markdown editor's folder.
  </Step>
  <Step title="Confirm">
    Click **OK**. Better Notes exports the note immediately and registers it for automatic syncing. Future changes on either side are handled automatically.
  </Step>
</Steps>

<Tip>To sync linked notes as a group, enable **Set Auto-Sync** while linked notes are included. Better Notes registers all linked notes to the same directory and syncs them together.</Tip>

## Setting the auto-sync period

You can configure how often Better Notes checks for changes:

1. Open **Edit** → **Preferences** (Windows/Linux) or **Zotero** → **Settings** (macOS).
2. Go to the **Better Notes** tab.
3. Set the **Auto-sync period** to the desired interval in minutes.

A shorter interval keeps files more current. A longer interval reduces background activity.

## The sync manager

The sync manager window shows all notes currently registered for sync, along with their last sync time and linked file path.

To open it: go to **Tools** → **Sync Manager** in the Zotero menu bar.

From the sync manager you can:

- **Refresh** — reload the list to reflect the current sync state.
- **Sync now** — force an immediate sync for selected notes.
- **Un-sync** — remove selected notes from the sync list (the `.md` file is not deleted).
- **Detect** — scan a folder to find `.md` files that already contain Better Notes metadata and register them automatically.

## The diff window

When both the Zotero note and the Markdown file have changed since the last sync, Better Notes opens a diff window instead of merging automatically. The diff window shows the two versions side by side with changes highlighted.

In the diff window you can:

- **Accept all** — keep the Markdown file's version.
- **Reject all** — keep the Zotero note's version.
- **Merge manually** — accept or reject individual change blocks.
- **Un-sync** — remove the note from sync and keep both versions as-is.
- **Skip** — postpone the merge until the next sync cycle.

After you resolve the diff, Better Notes writes the merged content to both the Zotero note and the `.md` file and updates the sync record.

## Markdown file structure

Synced Markdown files include a YAML front matter block generated by the `[ExportMDFileHeaderV2]` built-in template. By default, this block contains the note's tags, parent item title, and collection names, plus internal metadata that Better Notes uses to identify the file during detect scans.

You can customize the front matter by editing the `[ExportMDFileHeaderV2]` template in **Tools** → **Note Template Editor**. See [Note templates](/features/note-templates) for details.
