---
title: "Export notes from Zotero with Better Notes formats"
description: "Export Zotero notes to Markdown, DOCX, PDF, FreeMind, LaTeX, or a new Zotero note. Customize filenames and content with built-in templates."
---

Better Notes extends Zotero's note export with support for six output formats: Markdown, Microsoft Word (DOCX), PDF, FreeMind mind map, LaTeX, and a new Zotero note. Each format is accessible from the note editor toolbar, and Markdown and LaTeX outputs are customizable through built-in templates.

## Supported formats

<CardGroup cols={2}>
  <Card title="Markdown (.md)" icon="markdown">
    Plain-text Markdown with optional YAML front matter. Images can be embedded or saved as separate files in a linked attachment folder.
  </Card>
  <Card title="Microsoft Word (.docx)" icon="file-word">
    A formatted Word document suitable for sharing with collaborators who do not use Zotero.
  </Card>
  <Card title="PDF (.pdf)" icon="file-pdf">
    A rendered PDF of the note content.
  </Card>
  <Card title="FreeMind (.mm)" icon="diagram-project">
    A mind map file compatible with FreeMind and other tools that support the `.mm` XML format. Headings become nodes.
  </Card>
  <Card title="LaTeX (.tex)" icon="file-code">
    A `.tex` source file. Content is processed by the `[ExportLatexFileContent]` template, which you can customize to match your document class.
  </Card>
  <Card title="New Zotero note" icon="note-sticky">
    Creates a copy of the current note as a new item in your Zotero library, with all linked note content optionally embedded inline.
  </Card>
</CardGroup>

## How to export a note

<Steps>
  <Step title="Open the note">
    Open the note you want to export as a tab or in the item pane.
  </Step>
  <Step title="Open the export dialog">
    In the editor toolbar, click the **settings menu** (the gear icon) and select **Export**. If the note is already set up for Markdown sync, this menu item opens the sync info window instead — use the sync manager to export synced notes.
  </Step>
  <Step title="Select a format">
    The export options window lists all available formats. Select the format or formats you want. You can export to multiple formats in one step.
  </Step>
  <Step title="Choose a destination">
    A file or folder picker opens depending on the format. Select where to save the output and click **OK**.
  </Step>
</Steps>

## Markdown export options

Markdown export has additional settings for handling note links and images:

<AccordionGroup>
  <Accordion title="Embedded images">
    By default, images in the note are saved as separate files in a subfolder (configured via the **Sync attachment folder** preference). The Markdown file references them with relative paths.
  </Accordion>
  <Accordion title="Embedded note links">
    Enable **Embed linked notes** to replace each note link in the exported file with the full content of the linked note, produced by the `[QuickImportV2]` template. The output is a self-contained document.
  </Accordion>
  <Accordion title="Linked note files (standalone)">
    Enable **Export linked notes** to export each linked note as its own separate `.md` file alongside the main note, preserving the link structure.
  </Accordion>
  <Accordion title="YAML front matter">
    Enable **With YAML header** to prepend a YAML block generated by the `[ExportMDFileHeaderV2]` template. The default header includes tags, parent item title, and collections.
  </Accordion>
  <Accordion title="Auto-sync setup">
    Enable **Set Auto-Sync** to register the note (and any linked notes) for two-way Markdown sync. See [Markdown sync](/features/markdown-sync) for details.
  </Accordion>
</AccordionGroup>

## Customizing Markdown output with templates

Three built-in templates control the shape of exported Markdown files:

| Template | Controls | Edit via |
|----------|----------|----------|
| `[ExportMDFileNameV2]` | The filename of the exported `.md` file | **Tools → Note Template Editor** |
| `[ExportMDFileHeaderV2]` | The YAML front matter prepended to the file | **Tools → Note Template Editor** |
| `[ExportMDFileContent]` | Post-processing of the full Markdown content | **Tools → Note Template Editor** |

The default filename template produces a name like `My Note Title-ITEMKEY.md`, using the note title with special characters replaced by dashes, followed by the Zotero item key.

## LaTeX export

LaTeX export produces a `.tex` file. The content goes through the `[ExportLatexFileContent]` template before being written to disk, so you can add a `\documentclass` preamble, wrap content in environments, or strip elements you do not need.

When exporting multiple notes to LaTeX, you can choose to **merge** them into a single `.tex` file. The merge combines the notes in order and writes one output file.

<Tip>Edit `[ExportLatexFileContent]` in the Note Template Editor to add a full LaTeX document wrapper (`\documentclass`, `\begin{document}`, etc.) so the exported file compiles directly with `pdflatex` or `xelatex`.</Tip>

## Batch export via the API

Better Notes exposes an `exportNotes` function through its public API at `Zotero.BetterNotes.api.$export`. You can call this function from the Actions & Tags plugin or from a custom script to export multiple notes programmatically with any combination of format options.

```javascript
await Zotero.BetterNotes.api.$export.exportNotes(noteItems, {
  exportMD: true,
  withYAMLHeader: true,
  autoMDFileName: true,
  syncDir: "/path/to/output/folder",
});
```

See the [API reference](/api/export-import) for the full list of options accepted by `exportNotes`.
